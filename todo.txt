MAIL RECIEVING SYSTEM 
Create a folder called mailrx in the root of this project. It must be written in Go and implement a lightweight SMTP receiving server using github.com/emersion/go-smtp for the SMTP protocol and github.com/emersion/go-message for parsing raw email including attachments. At the RCPT TO phase, before accepting any email body, query the existing database using the project's established DB connection and ORM patterns to check if the recipient address exists as a valid customer email address — reject unknown addresses immediately with a 550 error, ensuring no spam or unknown mail ever reaches the queue. Once an address is validated, accept the email, parse the To: address and any attachments, then push a lightweight JSON message onto a queue for async processing. Check the migrations folder to understand the existing database schema and create any additional migrations needed to store inbound email records and valid recipient addresses following the same migration patterns already used in this project. The SMTP server config (port, domain, DB connection, queue DSN) must be driven by environment variables. The queue implementation should use whatever queue or messaging system is already present in this project, or default to Redis streams using github.com/redis/go-redis/v9 if none exists. Keep the mailrx package self-contained with its own server.go, handler.go, and config.go files.
In the main backend application, create a dedicated route following the existing routing patterns of this project that acts as the queue consumer and mail processor. This route or background worker must pull jobs off the queue, extract the customer from the To: address by looking up the database, retrieve any attachments, store documents using whatever file storage pattern already exists in the project, and update the relevant database records. The mailrx SMTP server and the backend processing route must be kept decoupled — mailrx only validates and enqueues, the backend route only processes — so each can be scaled and updated independently.
